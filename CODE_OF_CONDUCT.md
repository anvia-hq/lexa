# Code of Conduct

## Our Pledge

We want Lexa to be a respectful, practical, and welcoming project for contributors and users. Everyone participating in the project is expected to help maintain a constructive environment.

## Expected Behavior

- Be respectful and professional.
- Assume good intent, but be clear when something needs to improve.
- Keep feedback focused on code, behavior, and project outcomes.
- Welcome contributors regardless of background, identity, experience level, or location.
- Report issues in a way that helps maintainers understand and act on them.

## Unacceptable Behavior

- Harassment, threats, intimidation, or personal attacks.
- Discriminatory language or behavior.
- Sexualized language or unwanted attention.
- Deliberate disruption of discussions, issues, pull requests, or project spaces.
- Publishing private information without permission.

## Enforcement

Maintainers may remove content, close discussions, reject contributions, or restrict participation when behavior violates this code of conduct.

If you need to report a concern, contact the maintainers through the project's GitHub issue tracker or another project-maintainer channel. Reports will be reviewed with care and handled as privately as practical.

## Scope

This code of conduct applies to all project spaces, including the repository, issues, pull requests, discussions, and community channels associated with Lexa.
